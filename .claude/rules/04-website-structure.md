# Website Structure Rules

Before final implementation, always define:

- sitemap
- homepage section structure
- page roles
- CTA strategy
- future/coming-soon areas

## Current Page Direction
The website should support the following pages:

- Startseite
- Lösungen
- Branchen
- FAQ
- Über uns
- Kontakt
- Impressum
- Datenschutz

Chatbot and Dashboard may appear in the navigation, but must be presented honestly if they are still future-facing or in preparation.

## Homepage Default Structure
The homepage should usually follow this order:

1. Hero
2. Problem
3. Lösung
4. So funktioniert es
5. Funktionen / Vorteile
6. Branchen
7. Warum HG Automations
8. Zukunft / Erweiterbarkeit
9. FAQ
10. CTA
11. Kontaktformular

## CTA Direction
The primary CTA should be tightly aligned with automation value.

Preferred primary CTA:
- `Telefonservice automatisieren`

Secondary CTAs can include:
- `Mehr erfahren`
- `Kontakt aufnehmen`
- `Demo ansehen`

Do not use generic CTA wording if a more specific automation-focused option exists.
