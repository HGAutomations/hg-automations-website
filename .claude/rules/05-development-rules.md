# Development Rules

- Respect the existing Vite project structure
- Keep the code modular, readable, and maintainable
- Prefer a clean component structure over monolithic files
- Avoid unnecessary dependencies
- Build mobile-first
- Ensure strong responsiveness across common breakpoints
- Keep performance in mind from the start
- Prepare the site for deployment on Vercel
- Do not hardcode fake backend behavior
- Integrate the contact form in a production-ready way using the provided Web3Forms endpoint
- If secrets are present in plain files, recommend moving them to environment variables where appropriate
- Keep the implementation realistic and deployable today
